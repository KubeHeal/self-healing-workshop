= Self-Healing Workshop
:toc: left
:toclevels: 3

== Overview

This workshop demonstrates the **OpenShift AI Ops Self-Healing Platform** - an AI-powered self-healing solution for OpenShift clusters that combines deterministic automation with machine learning for intelligent incident response.

=== What You'll Learn

* **Module 0**: Introduction & Architecture - How the platform works
* **Module 1**: ML Model Training with Tekton - Train anomaly detection models
* **Module 2**: End-to-End Self-Healing with Lightspeed - Interactive AI-powered cluster management
* **Module 3**: OpenShift Lightspeed Deep Dive - Technical integration details

=== Prerequisites

* OpenShift 4.18+ cluster with admin access
* AgnosticD V2 configured (for deployment)
* Red Hat Demo Platform AWS sandbox (recommended)

== Getting Started

=== Option 1: Using AgnosticD V2 (Recommended)

The workshop includes an AgnosticD V2 workload that pre-deploys the entire platform:

[source,bash]
----
cd ~/agnosticd-v2

# Copy the workshop vars file
cp /path/to/self-healing-workshop/agnosticd/vars/self-healing-workshop.yml ../agnosticd-v2-vars/

# Provision the workshop environment
./bin/agd provision -g workshop1 -c self-healing-workshop -a sandbox1234
----

=== Option 2: Manual Deployment

If you prefer to deploy manually, follow the https://github.com/KubeHeal/openshift-aiops-platform[openshift-aiops-platform] README.
